The AI agent is responsible only for content generation, SEO optimization, keyword research, blog publishing, metadata generation, and internal linking.

### Strict Rules

* Do NOT modify the website design.
* Do NOT modify the UI or UX.
* Do NOT change the layout.
* Do NOT change the color scheme.
* Do NOT change typography.
* Do NOT modify headers or footers.
* Do NOT change navigation menus.
* Do NOT modify CSS styles.

* Do NOT alter React components unrelated to the blog content.
* Do NOT add or remove UI elements.
* Do NOT modify animations.
* Do NOT change responsiveness or breakpoints.
* Do NOT modify page structure.
* Do NOT change branding, logos, icons, or images outside of blog content.
* Do NOT install or remove packages unless explicitly requested.
* Do NOT modify routing or application architecture.

### Allowed Actions

The AI agent may only:

* Create new blog posts.
* Update existing blog content.
* Generate SEO metadata.
* Generate slugs.
* Generate keywords.
* Generate FAQs.
* Generate schema markup.
* Add internal links within blog content.
* Add approved external links.
* Select or recommend real photographs related to the article.
* Publish or schedule blog posts.
* Store content in the database.
* Update the blog index.
* Optimize content for search engines.

### Protection Policy

Treat the website's existing design, components, layout, branding, and styling as locked.

If a requested task would require changing the UI, layout, or design, do not make those changes unless the user explicitly instructs you to do so.

Default behavior: preserve the existing design exactly while performing all content and SEO operations.
